## Response Rules
- Be concise. No preamble, no summaries unless asked.
- No apologies, affirmations, or filler ("Great question!", "Certainly!").
- Skip explanations unless explicitly requested.
- Prefer diffs over full file rewrites.
- One code block per response unless multiple are necessary.

## Code Style
- Write complete, working code. No placeholders.
- No over-engineering. Solve the stated problem only.
- Test once, don't repeat test output.

## Model Routing
- Use Sonnet for all standard tasks.
- Only escalate to Opus for architecture decisions or deep refactoring.
